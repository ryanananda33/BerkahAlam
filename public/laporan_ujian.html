Berkah Alam

Konsep Dasar Pemrograman Web

Pembahasan teori konsep dasar CRUD, CMV (MVC), dan Keamanan Data pada Berkah Alam.

1. CRUD (Create, Read, Update, Delete)

CRUD adalah representasi dari fungsi-fungsi dasar manipulasi data pada penyimpanan persisten.

A. Create (Membuat Data Baru)

Proses menambahkan data baru ke dalam database. Pada project Berkah Alam:

  • Customer: Mengajukan order pembelian batu nisan dengan mengirim data ke OrderController@placeOrder yang menyimpannya ke tabel pesanan dan detail_pesanan.
  • Customer: Mengunggah bukti pembayaran yang kemudian disimpan ke tabel pembayaran.

B. Read (Membaca Data)

Proses menampilkan data yang ada pada database ke antarmuka pengguna.

  • Public: Melihat katalog produk nisan dan prasasti batu alam di Landing Page melalui LandingPageController@index.
  • Admin: Melihat tabel data transaksi pesanan masuk pada halaman Kelola Pesanan.

C. Update (Mengubah Data)

Proses memperbarui data yang tersimpan di database.

  • Customer: Mengubah alamat pengiriman dan nomor telepon pada profil akun mereka.
  • Admin: Melakukan konfirmasi pembayaran dengan mengubah kolom status pada tabel pembayaran dan pesanan menjadi diverifikasi atau ditolak.

D. Delete (Menghapus Data)

Proses menghapus data dari database secara permanen.

  • Admin: Menghapus kategori atau produk batu alam yang sudah tidak dijual dari sistem. Sistem menggunakan relasi ON DELETE CASCADE untuk menjamin data transaksi lama tidak bentrok.

2. CMV / MVC (Controller - Model - View)

Pola arsitektur penulisan program untuk memisahkan antara data, tampilan, dan logika bisnis utama.

A. Model

Merepresentasikan struktur data dan berhubungan langsung dengan tabel database MySQL melalui Eloquent ORM di Laravel.

Contoh: File app/Models/Produk.php mendefinisikan properti produk, relasi ke kategori (kategori()), dan detail pesanan.

B. View

Komponen antarmuka visual tempat data disajikan kepada pengguna, dibangun menggunakan Blade Templating Engine di Laravel.

Contoh: File resources/views/welcome.blade.php mengolah data array produk untuk ditampilkan dalam grid card produk.

C. Controller

Komponen penjembatan. Controller menerima request dari route web, memanggil fungsi Model untuk mengolah data database, dan mengirimkannya ke View untuk dirender.

Contoh: LandingPageController mengambil semua data produk aktif dari database dan mengirimkannya ke landing page.

3. Keamanan Data

Aspek perlindungan integritas dan privasi data aplikasi dari serangan siber:

  • Password Hashing: Menyimpan password dalam bentuk enkripsi satu arah (Bcrypt) sehingga jika database bocor, password tidak dapat dibaca.
  • CSRF Protection: Token pengaman unik otomatis pada setiap request form di Laravel dengan tag @csrf untuk mencegah manipulasi form dari luar aplikasi.
  • SQL Injection Prevention: Menggunakan Eloquent ORM yang memanfaatkan PDO Parameter Binding untuk memfilter seluruh input form sebelum masuk ke database.
  • Route Authorization: Penggunaan middleware auth dan role:admin untuk memblokir akses ke halaman admin dari pengguna umum atau peretas.

3. Script DDL Database (SQL Dump)

-- Pembuatan Tabel User
CREATE TABLE `users` (
  `id` bigint(20) UNSIGNED NOT NULL AUTO_INCREMENT,
  `name` varchar(255) NOT NULL,
  `email` varchar(255) NOT NULL,
  `password` varchar(255) NOT NULL,
  `role` varchar(255) NOT NULL DEFAULT 'customer',
  PRIMARY KEY (`id`),
  UNIQUE KEY `users_email_unique` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Pembuatan Tabel Produk
CREATE TABLE `produk` (
  `id` bigint(20) UNSIGNED NOT NULL AUTO_INCREMENT,
  `kategori_id` bigint(20) UNSIGNED NOT NULL,
  `nama` varchar(255) NOT NULL,
  `harga` decimal(15,2) NOT NULL,
  `stok` int(11) NOT NULL DEFAULT 0,
  `gambar` varchar(255) DEFAULT NULL,
  PRIMARY KEY (`id`),
  CONSTRAINT `produk_kategori_id_foreign` FOREIGN KEY (`kategori_id`) REFERENCES `kategori` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

FILE 3: Script dan UI Back End

Otorisasi Middleware, Logika Verifikasi Transaksi, CRUD Kategori/Produk, dan Ekspor Invoice PDF.

1. Logika Verifikasi Bukti Pembayaran (`AdminPesananController.php`)

Memproses keputusan admin untuk memverifikasi atau menolak bukti transfer yang diunggah customer.

public function verifyPayment(Request $request, $id)
{
    $order = Pesanan::with('pembayaran')->findOrFail($id);
    
    if (!$order->pembayaran) {
        return back()->with('error', 'Pesanan ini belum memiliki bukti pembayaran.');
    }

    $request->validate([
        'action' => ['required', 'in:verify,reject'],
    ]);

    if ($request->action === 'verify') {
        $order->pembayaran->update(['status' => 'diverifikasi']);
        $order->update(['status' => 'diverifikasi']);
        return back()->with('success', 'Pembayaran berhasil diverifikasi!');
    } else {
        $order->pembayaran->update(['status' => 'ditolak']);
        $order->update(['status' => 'ditolak']);
        return back()->with('success', 'Pembayaran ditolak!');
    }
}

2. Ekspor Struk Invoice ke PDF (`dompdf`)

Logika backend memanggil file template struk dan merendernya ke dokumen PDF.

use Barryvdh\DomPDF\Facade\Pdf;

public function exportPdf($id)
{
    $order = Pesanan::with(['user', 'pembayaran', 'detailPesanan.produk'])->findOrFail($id);
    $pdf = Pdf::loadView('admin.pesanan.pdf', compact('order'));
    return $pdf->download('Invoice-BA-' . $order->id . '.pdf');
}

3. CRUD Produk dengan Upload Gambar (`AdminProdukController.php`)

Menyimpan data produk baru serta mengunggah gambar produk secara aman ke public server.

public function store(Request $request)
{
    $request->validate([
        'kategori_id' => 'required|exists:kategori,id',
        'nama' => 'required|string',
        'harga' => 'required|numeric',
        'gambar' => 'required|image|mimes:jpeg,png,jpg|max:2048',
    ]);

    $data = $request->all();

    if ($request->hasFile('gambar')) {
        $file = $request->file('gambar');
        $filename = time() . '_' . $file->getClientOriginalName();
        $file->move(public_path('uploads/produk'), $filename);
        $data['gambar'] = 'uploads/produk/' . $filename;
    }

    Produk::create($data);
    return redirect()->route('admin.produk.index')->with('success', 'Produk ditambahkan!');
}
