Laporan teknis ini disusun untuk mendokumentasikan hasil analisis fungsionalitas sistem (melalui model Use Case) dan rancangan logikal basis data (melalui Entity Relationship Diagram beserta Kamus Data) pada website pemesanan kerajinan batu alam kustom "Berkah Alam".

1. Analisis Peran Aktor (Actors)

Sistem ini beroperasi dengan membedakan hak akses dan perilaku navigasi bagi dua aktor utama yang diidentifikasi di bawah ini:

Customer (Pelanggan)

Pengguna terdaftar. Berhak melakukan pendaftaran akun (register), login ke sistem, melakukan transaksi pemesanan produk nisan/prasasti kustom, serta memantau riwayat dan status pesanan.

Admin (Administrator)

Pengelola internal toko/workshop. Memiliki wewenang login sistem, mengontrol data produk & kategori, CMS konten hero/galeri/testimoni, mengelola transaksi pesanan, memverifikasi pembayaran, serta mencetak invoice PDF.

Kamus Data & Tabel Spesifikasi

3.1. Tabel users (Master User)

Kolom Tipe Data Atribut Keterangan
id bigint(20) unsigned Primary Key (AI) Kunci utama pengenal user.
name varchar(255) - Nama lengkap pengguna.
email varchar(255) Unique Key Surel unik login sistem.
password varchar(255) - Password di-hash Bcrypt.
role varchar(255) - Role pengguna: `admin` / `customer`.
phone varchar(20) - Nomor kontak/WhatsApp.
address text - Alamat domisili pengiriman.

3.2. Tabel kategori (Kategori Master)

Kolom Tipe Data Atribut Keterangan
id bigint(20) unsigned Primary Key (AI) ID pengenal unik kategori.
nama varchar(255) - Nama jenis produk (cth: "Batu Nisan").

3.3. Tabel produk (Katalog Produk)

Kolom Tipe Data Atribut Keterangan
id bigint(20) unsigned Primary Key (AI) ID pengenal produk.
kategori_id bigint(20) unsigned Foreign Key Merujuk pada tabel `kategori.id` (Cascade).
nama varchar(255) - Nama kerajinan batu nisan/prasasti.
harga decimal(15,2) - Harga per pcs.
stok int(11) - Sisa stok di workshop.
deskripsi text - Spesifikasi ukuran dan tipe batu alam.
gambar varchar(255) - Path file upload gambar nisan.

3.4. Tabel pesanan (Transaksi Header)

Kolom Tipe Data Atribut Keterangan
id bigint(20) unsigned Primary Key (AI) No Nota / Invoice.
user_id bigint(20) unsigned Foreign Key Merujuk pada tabel `users.id` (Cascade).
tanggal date - Tanggal order masuk.
total decimal(15,2) - Total tagihan belanja.
status enum(...) - `pending`, `diverifikasi`, `diproses`, `selesai`, `ditolak`.

3.5. Tabel detail_pesanan (Transaksi Detail & Kustom)

Kolom Tipe Data Atribut Keterangan
id bigint(20) unsigned Primary Key (AI) ID baris detail.
pesanan_id bigint(20) unsigned Foreign Key Merujuk pada `pesanan.id` (Cascade).
produk_id bigint(20) unsigned Foreign Key Merujuk pada `produk.id` (Cascade).
qty int(11) - Kuantitas belanja.
harga decimal(15,2) - Harga barang sewaktu dibeli.
catatan_ukiran text - Tulisan yang dipahat di atas produk.

3.6. Tabel pembayaran (Upload Bukti Bayar)

Kolom Tipe Data Atribut Keterangan
id bigint(20) unsigned Primary Key (AI) ID bukti pembayaran.
pesanan_id bigint(20) unsigned Foreign Key Merujuk pada `pesanan.id` (Cascade, 1:1).
bukti_pembayaran varchar(255) - Path file bukti foto transfer.
tanggal_bayar datetime - Waktu pengunggahan file.
status enum(...) - Status pembayaran: `pending`, `diverifikasi`, `ditolak`.

3.7. Tabel Pendukung CMS & Landing Page

  • hero: Konfigurasi banner judul dan gambar background promo di halaman utama.
  • galeri: Dokumentasi visual portofolio pemahatan batu nisan.
  • testimoni: Ulasan kepuasan dan rating 1-5 bintang dari pelanggan.

4. Integritas Hubungan Data

Integritas relasional terjamin melalui aturan ON DELETE CASCADE pada foreign key constraints:

  1. Apabila data users dihapus dari database, seluruh riwayat belanja pada tabel pesanan otomatis dibersihkan.
  2. Apabila baris pesanan dihapus oleh admin, rincian kustom pada detail_pesanan serta file bukti transfer pada tabel pembayaran ikut terhapus di database.
  3. Penghapusan master kategori otomatis melenyapkan data produk di dalamnya demi menjamin tidak adanya rekaman yatim piatu di tingkat DBMS.
